disease_reports — the only table in the database
📋 disease_reports
🔑 id INTEGER Auto-incremented primary key. Unique ID for every disease report. PKAUTOINCREMENT
🌿 disease_type TEXT Name of the detected plant disease (e.g. "Leaf Blight"). NOT NULL
📊 confidence TEXT AI model's confidence score as text (e.g. "92.3%"). OPTIONAL
💊 remedy TEXT Recommended treatment / remedy for this disease. OPTIONAL
🛡️ prevention TEXT Preventive measures to avoid future outbreaks. OPTIONAL
📍 latitude REAL GPS latitude of the farm where report was filed (e.g. 28.6139). OPTIONAL
📍 longitude REAL GPS longitude of the farm where report was filed (e.g. 77.2090). OPTIONAL
🕐 timestamp TEXT UTC ISO-8601 datetime when the report was saved (e.g. "2026-03-30T10:00:00"). NOT NULL
Why these columns?
disease_type + timestamp — core required data. Every analysis must have these two.
confidence + remedy + prevention — AI output fields, nullable because early versions may skip them.
latitude + longitude — optional geo-tagging. Null = farmer didn't share location. When present, enables the get_nearby_outbreak_risk() geo-query.
id — auto-increment keeps inserts simple; no need to generate IDs externally.
All functions defined in database.py
INIT get_connection() → sqlite3.Connection
Opens a connection to cropradar.db. Sets row_factory = sqlite3.Row so every fetched row behaves like a dictionary (access by column name instead of index).
INIT init_db() → None
Called once at startup. Runs the CREATE TABLE IF NOT EXISTS SQL to ensure the disease_reports table exists before any reads or writes.
WRITE insert_report(disease_type, confidence, remedy, prevention, latitude?, longitude?) → int
Inserts one new disease report. Auto-generates a UTC timestamp. Returns the new row's id so the caller knows which record was created.
READ get_all_reports() → list[dict]
Returns every row in the table, sorted newest-first. Used for dashboards and full history views.
READ get_recent_reports_by_disease(disease_type, hours=48) → list[dict]
Filters reports to a specific disease within a time window (default 48 h). Useful for tracking how many times "Leaf Blight" was reported this week.
READ get_outbreak_diseases(threshold=3, hours=48) → list[dict]
Outbreak detector. Groups all reports by disease, counts them within the time window, and keeps only those reaching the threshold. Returns [{disease_type, count}, ...] sorted by severity.
GEO _haversine_km(lat1, lon1, lat2, lon2) → float
Private helper. Calculates the great-circle distance in km between two GPS coordinates using the Haversine formula. No external libraries needed — pure math.
GEO get_nearby_outbreak_risk(lat, lon, radius_km=50, threshold=3, hours=48) → list[dict]
Geo-aware outbreak detector. Fetches recent geo-tagged reports, computes distance using Haversine for each row, and counts only those within radius_km. Returns diseases at risk near a given farm location.
